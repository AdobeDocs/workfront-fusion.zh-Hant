# 使用執行一次來測試案例

建立、更新或修訂案例時，您可以使用「執行一次」按鈕來依需求觸發案例。 如此一來，您便可測試情境，而不需等候其觸發邏輯，例如特定事件或輪詢間隔。

您也可以執行案例一次，以設定資料結構

>[!TIP]
>
>建議您經常在建立和調整情境時進行測試。

## 使用執行一次功能

在情境編輯器中找到「執行一次」功能。

1. 按一下左側面板中的&#x200B;**[!UICONTROL 案例]**&#x200B;索引標籤。
1. 選取您要執行「案例評分專家」的案例。
1. 按一下情境上的任何位置，以輸入情境編輯器。
1. 按一下「情境編輯器」左下角的&#x200B;**執行一次**。
1. （條件式）如果案例是由webhook觸發或為子案例，請選取測試回合的輸入。

   * **等候外部webhook呼叫**：此案例將開始接聽傳入的webhook呼叫。 您必須從外部系統觸發webhook，才能提供webhook。

     例如，如果webhook在Workfront中接聽新請求，您必須在Workfront中建立新請求，以觸發它完成「執行一次」功能。

     >[!NOTE]
     >
     >此選項僅適用於webhook案例。

   * **使用先前的執行輸入**：您必須選取先前執行。 「執行一次」執行會使用觸發所選執行的輸入資料。

     若要選取上一個執行，請選取該執行並按一下&#x200B;**執行**。

     若要在選取執行之前檢查輸入資料，請按一下該執行的&#x200B;**預覽輸入**。

     >[!NOTE]
     >
     >案例必須完成先前的執行，此選項才可用。

   * **手動提供輸入**：您必須為案例輸入提供webhook裝載。 這必須是JSON格式。

     若要提供輸入，請在&#x200B;**Webhook承載**&#x200B;方塊中輸入文字，然後按一下&#x200B;**執行**。
